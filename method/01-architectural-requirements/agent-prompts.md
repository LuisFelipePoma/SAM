# Agent Prompts - Architectural Requirements

## Intake

```text
Lee el brief del cliente. Extrae objetivo de negocio, alcance, stakeholders, restricciones, sistemas externos, glosario, riesgos y preguntas abiertas. No propongas arquitectura todavia.
```

## ASR

```text
Detecta los requisitos arquitectonicamente significativos. Clasificalos como funcional clave, atributo de calidad, restriccion o concern. Marca ambiguedades y requisitos no medibles.
```

## QAW-lite

```text
Convierte los atributos de calidad en escenarios medibles. Luego propone una priorizacion por importancia de negocio, dificultad tecnica y riesgo. El arquitecto debe aprobar la lista de drivers primarios.
```

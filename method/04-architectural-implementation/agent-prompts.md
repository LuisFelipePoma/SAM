# Agent Prompts - Architectural Implementation

## Implementation Plan

```text
Con architecture-document.md, design-decisions.md y el input de fase 4, genera implementation-plan.md usando method/04-architectural-implementation/template.md. Divide el trabajo en slices pequenos. Cada slice debe tener driver/ADR, story, criterios de aceptacion, tests minimos y constraints arquitectonicos.
```

## Design System

```text
Si el proyecto tiene frontend, genera design-system.md usando method/04-architectural-implementation/design-system-template.md. Incluye principios UI, tokens, colores, tipografia, spacing, componentes, formularios, tablas/listas, estados, accesibilidad, responsive e icon/library policy.
```

## Code Agent Slice Prompt

```text
Prepara un prompt para implementar solo un slice. Incluye contexto minimo, drivers/ADRs relacionados, archivos/modulos esperados, criterios de aceptacion, tests minimos y constraints. Prohibe implementar slices no solicitados.
```

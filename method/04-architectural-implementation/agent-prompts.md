# Agent Prompts - Architectural Implementation

## Implementation Plan

```text
Using architecture-document.md, design-decisions.md, and the phase 4 input, generate implementation-plan.md with method/04-architectural-implementation/template.md. Divide the work into small slices. Each slice must have driver/ADR, story, acceptance criteria, minimum tests, and architectural constraints.
```

## Design System

```text
If the project has a frontend, generate design-system.md using method/04-architectural-implementation/design-system-template.md. Include UI principles, tokens, colors, typography, spacing, components, forms, tables/lists, states, accessibility, responsive behavior, and icon/library policy.
```

## Code Agent Slice Prompt

```text
Prepare a prompt to implement only one slice. Include minimum context, related drivers/ADRs, expected files/modules, acceptance criteria, minimum tests, and constraints. Forbid implementing unrequested slices.
```
